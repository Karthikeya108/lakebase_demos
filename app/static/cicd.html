How it works

Each environment is a Lakebase branch. Migrations are defined as a changelog and tracked per-environment via a schema_migrations table — just like Liquibase or Flyway.

Apply migrations to dev first, validate, promote to staging, then production. Each branch is fully isolated — schema changes on dev don't affect production until promoted.
