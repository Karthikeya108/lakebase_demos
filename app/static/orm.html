ORM (SQLAlchemy) Demo

SQLAlchemy 2.x with Lakebase Postgres — auto-reflection, filtered browsing, relationship navigation, and ORM vs raw SQL comparison

How it works

Lakebase is standard PostgreSQL, so any ORM works out of the box. This demo uses SQLAlchemy 2.x with the postgresql+psycopg driver and OAuth token authentication.

Tables are auto-reflected from the database — no model definitions needed. Foreign keys, types, and constraints are all discovered dynamically.

Auto-Reflected Tables

MetaData.reflect(bind=engine) — SQLAlchemy inspects Postgres system catalogs to discover tables, columns, types, PKs, and foreign keys.
