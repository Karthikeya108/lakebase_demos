Row-Level Security Demo

Same query, different results — Postgres RLS filters rows based on who you are

How it works

Each regional manager has a Postgres role (rls_northeast, rls_west, etc.) with an RLS policy:

CREATE POLICY region_filter ON agents USING (LOWER(region) = REPLACE(current_user, 'rls_', ''))

The app runs SET ROLE rls_<region> before querying. The SQL is always SELECT * FROM agents — Postgres handles the filtering.
