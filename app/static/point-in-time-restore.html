Point-in-Time Restore Demo

Simulate a disaster, then recover data by branching from a point in time

How it works

Lakebase automatically retains history for up to 35 days. No checkpoints or snapshots needed — you can branch from any past timestamp within the retention window.

CREATE BRANCH restored FROM production AT TIMESTAMP '2026-03-09 14:30:00'

The restored branch is a full copy-on-write snapshot — reads are instant, and production is unaffected.
